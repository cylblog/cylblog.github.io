并查集

Posted on 2021-06-17

基本概念

并查集算法在很多算法中都会简单的涉及，比如最小生成树的kruskal算法等。

并查集是一种树形的数据结构，并查集也称为不相交集数据结构，可以用于处理一些不相交集合的合并及查询问题，是一种用来管理元素分组情况的数据结构。“并”“查”“集”这三个字分别取自Union（合并）、Find(查询)、Set（集合）这三个单词。也就是说，并查集支持合并及查询操作。

并查集两个主要操作：
1.合并（Union）：把两个不相交的集合合并为一个集合。
2.查询（Find）：确定元素属于哪一个集合，可以用来判断两个元素是否在同一个集合中

并查集是通过一个一维数组f[]来实现，其中 f[i] 表示元素i的父结点，而父结点本身也是这个集合中的元素。例如：f[1] = 2，表示元素1的父结点是元素2，即元素1和元素2属于同一个集合。另外，如果f[i] == i, 则说明元素i是该集合的根结点，对于同一个集合来说有且只有一个根结点，故可将其作为所属集合的标识。

我们可以举个例子来理解一下并查集的应用

题目背景
若某个家族人员过于庞大，要判断两个是否是亲戚，确实还很不容易，现在给出某个亲戚关系图，求任意给出的两个人是否具有亲戚关系。

题目描述
规定：x和y是亲戚，y和z是亲戚，那么x和z也是亲戚。如果x,y是亲戚，那么x的亲戚都是y的亲戚，y的亲戚也都是x的亲戚。

输入格式
第一行：三个整数n,m,p，（n<=5000,m<=5000,p<=5000），分别表示有n个人，m个亲戚关系，询问p对亲戚关系。

以下m行：每行两个数Mi，Mj，1<=Mi，Mj<=N，表示Mi和Mj具有亲戚关系。

接下来p行：每行两个数Pi，Pj，询问Pi和Pj是否具有亲戚关系。

输出格式
P行，每行一个’Yes’或’No’。表示第i个询问的答案为“具有”或“不具有”亲戚关系。

题目来源：https://www.luogu.com.cn/problem/P1551

对于这个问题，我们就可以考虑用并查集将所有人划分到若干个不相交的集合中，每个集合里的人彼此是亲戚，为了判断两个人是否是亲戚，则只需判断他们是否属于同一个集合即可。

基本操作

总的来说，并查集的使用需要先初始化f[]数组，然后再根据需要进行查询或合并的操作。

1.初始化
一开始，我们认为每个元素都是独立的一个集合，因此需要令所有的f[i] = i 。

2.查询
由于规定同一个集合中有且只有一个根结点，因此查询操作就是对给定的结点寻找其根结点的过程，即反复寻找父结点，直到找到根结点（即f[i] == i的点)，可以用while循环或者递归实现。判断两个元素是否属于同一个集合，只需要看他们的根结点是否相同即可。

路径压缩

上面讲解的并查集查询函数是没有经过优化的，当题目给出的元素数量很多并且形成一条链时，这个查询函数的效率是非常低的。

那么如何优化查询操作呢？这里我们可以使用路径压缩，路径压缩是一种在执行“查找”时扁平化树结构的方法。既然我们只关心一个元素对应的根结点，那我们希望每个元素到根结点的路径尽可能短，最好只需要一步，关键就在于路径上的每个结点都可以直接连接到根上。
