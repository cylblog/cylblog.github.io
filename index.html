重拾排序算法

发表于 2022-03-13 分类于 算法

冒泡排序

稳定性：在相邻元素相等的情况下不会交换位置，所以冒泡排序时稳定的。

平均时间复杂度：O(n²)

选择排序

稳定性：选择排序是不稳定的。

平均时间复杂度：O(n²)

插入排序

稳定性：不大于当前数的位置并不需要交换，因此是稳定的。

时间复杂度：O(n²)

希尔排序

在基于插入排序的基础上提出了希尔排序：

  1. 插入排序在数组基本有序的情况下，可近似达到O(n)复杂度，效率高。
  2. 插入排序每次只能将数据移动一次，在数组较大且基本无序的情况下性能会迅速恶化。

稳定性：不稳定。

平均时间复杂度：O(nlogn)

归并排序

稳定性：稳定。

平均时间复杂度：O(nlogn)

快速排序

稳定性：不稳定。

平均时间复杂度：O(nlogn)

监听数组

我们可以看到数组的一些方法例如pop、push为数组添加或者删除元素时，不能触发属性arr的setter，无法对属性arr中元素进行实时监听，为此vue2重写了这些数组。

同时我们注意到的上面的push、pop直接通过索引为数组添加元素时会触发arr的getter方法，这是由于这些操作的返回值。比如使用push为数组添加元素时，push方法返回值总是添加之后数组的新长度，当访问数组新长度时自然会触发settet方法。

Vue3响应式原理

监狱vue2中监听数据时的一些弊端，vue3监听数据的时候弃用了Object.defineProperty()，而使用了Proxy()。

语法

const p = new Proxy(target,handler)

参数：

  • target：代理的对象
  • handler：一个以函数为属性的对象，其中定义了代理对象的各种操作

注意：set函数必须返回一个布尔值表示设置成功，返回false则报错TypeError。

与Object.defineProperty()差别：

  1. Proxy代理的是整个对象，而不是对象的某个特定属性。
  2. 代理之后操作的是实例对象，而不是原对象。

什么是跨域

跨域是由于浏览器的同源策略，为了防范跨站脚本的攻击，禁止客户端对不同域下的文档或脚本进行跨站调用资源。

相反，同源就是指协议、域名、端口号相同。

同源是浏览器的安全功能，如果没有这个安全策略，浏览器很容易受到XSS、CSFR攻击。

重绘和重排

发表于 2022-02-13

前置知识

  1. 浏览器使用流式布局。
  2. 浏览器会把HTML解析成DOM，把CSS解析成CSSDOM，DOM和CSSDOM合并就产生了Render Tree。
  3. 有了Render Tree，我们就知道了所有节点的样式，然后计算他们在页面上的大小和位置，最后把节点绘制到页面上。
  4. 由于浏览器使用流失布局，对Render Tree的计算通常只需要计算一遍即可，但table及其内部元素除外，他们可能需要多次计算，通常需要花3倍于同等元素的时间，这也是为什么避免使用table布局的原因之一。

什么是重绘和重排

  • 重绘：当一个元素的外观发生改变，但没有改变布局，重新把元素外观绘制出来的过程。

  • 重排：当DOM的变化影响了元素的在浏览器中的位置时，浏览器重新计算元素的位置，将其放置在正确的位置的过程叫做重排。

    仅仅改变元素的外观，肯定不会引起网页重新生成布局，但浏览器完成重排之后，将会重新绘制受到此次重排影响的部分。

    重绘不一定会引起重排，但是重排一定会出现重绘。

​

如何触发重绘和重排

任何改变用来构建渲染树的信息都会导致一次重绘或重排。

  1. 通过visibility：hidden隐藏一个DOM节点会触发重绘，因为元素继续占有原来的位置。
  2. display：none会触发重排，因为元素不再占有原来的位置。
  3. 调整窗口的大小、改变字号、滚动引起重排。
  4. 添加或删除可见的DOM属性引起重排。
  5. 元素尺寸或位置发生改变引起重排。
  6. ……

如何避免重绘或重排

  1. 集中改变样式，不要一条一条地修改DOM的样式。
  2. 不要把DOM节点的属性值放在循环里当成循环里的变量。
  3. 将动画效果应用到使用position为fixed或absolute的元素上。
  4. 不使用table布局，因为可能很小的一个改动会造成整个table的重新布局。
  5. 动画开启GPU加速，translate使用3D变化。
  6. 尽可能在DOM树的最末端改变样式。
  7. …
