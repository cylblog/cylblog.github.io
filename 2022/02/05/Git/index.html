代码提交规范

master分支

主分支，只能从release分支或hotfix分支并入。

develop分支

开发分支，包含即将发布的代码

release分支

发布分支，只能接受从develop分支的代码，正式发布前要将代码提交到该分支，如果发现问题可直接在该分支修改，测试无误后合并入master分支以及develop分支。

feature分支

新功能分支，名称不固定，通常为新功能的名字，一般一个新功能对应一个分支，并且都是基于develop分支创建。例如：有一个新的功能登录，这个时候就可以从develop分支创建一个login分支进行登录功能的开发。开发完成后并入develop分支。

hotfix分支

紧急修复分支，用于紧急修复bug时使用。

开发按流程

  1. 新建空白文件，打开终端，初始化本地仓库git init，再克隆远程仓库代码git clone ......
  2. 若想要的分支不存在，那么输入git checkout -b develop创建并切换到develop分支，该指令相当于git branch develop再git checkout develop；如果该分支已经存在，并且想要直接在该分支上修改代码，那么直接切换到该分支即可
  3. 基于develop分支创建新功能分支并切换到该分支git checkout -b [feature_name]
  4. 将开发完毕的代码提交到从工作区暂存到缓存区git add .，然后再提交git commit -m 'message',提交到本地仓库
  5. 新功能开发完毕之后合并会原来的分支，首先切换到原来分支git checkout [name]，拉取原来分支代码git pull origin [name]，再进行合并git merge [name]合并develop分支到当前分支，提交到远程仓库git push origin [仓库名]
  6. 准备发布时，从develop分支提交PR合并到release分支。
  7. 测试代码，修复bug。
  8. 将测试无误的代码提交PR从release分支合并到develop分支以及master分支。

其它指令

  • 删除本地分支：git branch -d [name]

  • 删除远程分支：git push origin --delete [name]

  • 查看日志：git log

  • 查看分支：git branch（包括本地和远程）

  • 清理本地无效分支(远程已删除本地没删除的分支): git fetch -p

  • 切换分支的另一个指令git switch [name]

  • 撤销指令：git reset --soft/mixed/hard
